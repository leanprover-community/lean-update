# create-release-note

Use this skill when writing release notes.

## Basic Guidelines

- Do not mention refactoring in release notes.
  Only describe changes that matter to users, such as added features or bug fixes.
- If there are breaking changes, explain all of them in a `Breaking Changes` section and describe how users can adapt to those changes.
- If a completed and closed issue is related to a change, include a link to that issue in the release notes.
- End the release notes with `Full Changelog: {link to commits between tags}`.
